Either way, web adds these over TUI:

💬 Hover tooltips
Key hints on hover, not just status bar
📊 Visual progress bars
Gradient bars, not just % text
🔗 Clickable session logs
Open log file inline in panel
⌨️ Keyboard still works
All TUI shortcuts still function
📱 Responsive
Adapts between desktop and mobile breakpoints

My recommendation: B (master/detail) — gives the web version a clear identity beyond "TUI in a browser", while staying structurally identical.
