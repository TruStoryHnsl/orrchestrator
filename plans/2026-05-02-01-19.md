CONCORD-EXTENSIONS - PLAN

These are all the extensions that I would like to ship with concord when the full version goes live.

# Design Considerations for ALL Extensions #
- extensions need to be explicit about the UX mode in which they operate.
    - some modes include:
        - Party Mode: TV or desktop hosts the app with one display and mobile connection to the app display something else. Usually user-specific UI    
        - Display Mode: Standard shared display where all users see the same window
        - Service Mode: Displays a unique instance to every user that connects
        - Chat mode: Displays inside a texh chat channel and is interactable primarily via a chatroom. Basic party style games can be quickly thrown together via the chat game engine that we are building aloingside concord. The intent here is that users can create very simple and straightforward bot subrouitines that allow the bot to administer any kind of game the user wants to play.
        - Hybrid Mode: One level of complexity higher than the chat mode. It splits the screen so there is a media display and a chatroom usable at once. This makes more involved story based chat games possible.
