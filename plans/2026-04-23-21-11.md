ORRBEAM - New Design Goals

- I want orrbeam to be able to connect to many different nodes. Each node should have its own list of known connections.

- I want to somehow utilize this to allow two players to interact with one instance of a game simultaneously. this is achieved by the host allowing the remote user to control the host computer in parallel with the host. 

    - How do you handle two keyboards? two mice? 
    - The primary audience here is the emulation crowd. I want to be able to play emulated couch co-op games with friends. 
    - How many instances can we connect at once? 
    - if its too much processing work can we use my server as a bridge to increase concurrent users?
