ORRCHESTRATOR_BETA_REDESIGN

A full service ai powered software development hypervisor that unifies my ai workflow and enables me to design complicated and thorough node-based corporate emulation models for ai agents. 

# UX #
 
- the user can be desiging and testing constantly while the development happens automatically. This allows the user to simultaneously manage the development of several different software solutions like a hypervisor. 
- The user can also create workflow templates that define how ai development teams organize their agents into different workflows that are optimized for different kinds of projects.
- A user can choose to use a simple development workflow with only a supervisor and developer agent if they need to make a single function bash script, or perform some basic system configuration change. 
- Or they can choose a dev workflow that utilizes 4 layers of management hierarchy and multiple corporate style departments that each have teams of agents that each have a particuler domain of the development plan that is theirs to execute and particular instruction on how best to go about it. This kind of detailed and thorough workflow is well suited for large software applications. 
- the user can also choose to create their own dev workflow or modify an existing one in an integrated node based ai labor design tool.
- I want to be able to submit a detailed plan for an application that orrchestrator then uses to fascillitate an ai managed software dev pipeline.  
- I want to pump out new instructions with no regard for pacing because the every new submission is processed and routed to the right project
- the project teams process those queued feedback packages into items on the development map.  
- couple that with tmux controlled local claude code sessions and you have a potentially fully autonomous software development pipeline that completes itsself with minimal user input. 
- dynamically throttle the pace of development with respect to the usage limits my available models are subject to.
- can use model API's from all the primary distributors as well as ollama models. 
- monitors API usage and stops sending new 
- uses layered processes of abstraction and recompilation to minimize the token length of the executable dev prompts 



# AGENT LIBRARY # 

	## Admin Department ## 
		
	- Executive Assistant: default interface with user. It passes any software devleopmnent related input to the chief operations officer. 
	- Chief Operations Officer: processes software development related prompts from the user into concise de-duplicated token efficency optimized prompts that are distributed to the job queues of the appropriate project.
	- Intelligence Resources Manager: watches API usage of relevant ai models. decides wether a project is allowed to continue working through its queue or wether it needs to stop and wait for API resources to free up. 
	- Mentor: advises the user on strategy decisions. 

	## Development Department ##

		### Leadership ### 
		- Project Manager: oversees the development process in a heuristic loop of plan, build, test, break, plan, build, test, break, and so on. This loop ensures that the team iterates on their work until the whole team is satisfied with the deliverable theyve been asked to create. Maintains a broad view and structurally optimizes the development process. Reccomends agent team configurations. Instructs the Project Coordinator of what needs to be accomplished at every step in development.
		- Project Coordinator: processes instructions from the manager, delegates labor to agents, and suggests skills for them to use to execute the instruction. has department specific identity that allows it to recognize when an instruction is in the wrong place. It is aware of other projects and their basic featuresets and it looks for opportunities to repurpose code to minimize token use.  
		- Talent Scout: reads the instructions and creates specialty agents who are instructed to work with a research agent compile a comprehensive knowledge set of a narrow topic. Maintains a database of specialist agents for reuse later.
   
		### Engineering ### 
		- Software Engineer: works with the researcher to design the optimal implementation of a requested instruction. Maintains the development roadmap for the project as instructions and designs come in. Maintains a broad perspective of the application. 
		- Developer: implements software solutions according to the supervisors instructions
		- Feature-Tester: checks the work of the developer agent and creates tools like VM's, Playwright, cellular hotspots for network seperation, etc. that allow it to test features in production envornments. Intelligently uses local machines with enabled ssh controls to replicate real world use cases. The feature-tester knows to never ever sign off a feature that has not been tested in a deployment environment. If the feature-tester isnt capable of testing something it will design a guide for the user to do it theirself and report the results to claude. 
		- Researcher: conducts comprehensive research on software engineering solutions to create accurate and up to date reports of the state of the software ecosystem the dev team is working within. 
		- UI Designer: designs useful, fast, and beautiful interfaces that facilitate the US design established by the user.
		- Specailist: Works with a research agent to update their agent file with comprehensive knowledge of a narrow topic.  

		### Quality Assurance ###
		- Penetration Tester: attempts to compromise the security of the program and reports on the results of its effors. 
		- Beta-Tester: tries to break things to find failure points and report them. 

		### Dev-Ops ###
		- Versioning Manager: Uses tools like the git MCP server to keep the project backed up, organized, and semantically versioned. submits update in well packeged commits.
		 


	## Marketing Department ##
	- UX Specialist: audits the applications interface design across all platforms slated for distribution and writes a report on the the current state of the interface and how it could improve. 
	- Market Researcher: conducts a thorough online investigation into the target market of a software package. 


	## Legal Department ##
	- Licensing Auditor: analyzes the project for dependencies and creates a report of legal 
	- Copyright Investigator: researches any relevant copyrights and reports on any conflicts with the development plan.
